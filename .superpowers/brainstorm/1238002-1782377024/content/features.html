What should the dashboard manage?

Currently the dashboard has 7 read-only pages. Only Keys & Budgets are interactive. Here's what "CLI parity to WebUI" means — pick what matters most to you.

A

Provider Management (CRUD)

Add, edit, remove providers from the UI — no more editing config.yaml. Toggle providers on/off. Test connections. Currently read-only.

B

Combo Editor (CRUD)

Create, edit, delete combos from the UI. Reorder models with drag-and-drop. Currently read-only.

C

Token Saver Toggles

Toggle RTK/Caveman/Ponytail on/off from the UI. Set Ponytail level. Currently config-file-only.

D

Server Settings Panel

Edit server settings (port, host, require_api_key) and view config from the UI. Currently config-file-only.

E

Pricing Overrides

Add/edit custom model pricing from the UI. Currently CLI-only (janus pricing list/show), no add/edit.

F

CLI Tool Setup Helper

One-click configure Claude Code, Codex, Cursor, Cline — shows the exact env vars / commands to copy. (9Router's most popular feature)
